## 作戰分析

### 關鍵洞察

1. **評分標準的真相**：他們明確說「不期待全部完成」，重點是：

   - **Discovery**：主動釐清需求的能力
   - **Formulation**：規格化能力
   - **Automation**：AI 輔助開發能力
   - **多次 Release**：敏捷迭代能力

2. **致命要求**：

   - **必須主動提問**（不提問直接落選）
   - **必須有 E2E 測試**
   - **必須多次 Release**

3. **技術債風險**：
   - 需要熟悉 Spring Boot + Next.js
   - 時間只有 10 天
   - 還要學 E2E 測試

## 核心策略：聰明的取捨 + AI 最大化

### 策略原則

1. **不要做完美主義者**：功能「能動」比「完美」重要
2. **善用 AI**：讓 Claude/GPT 寫 80% 的 boilerplate code
3. **刷存在感**：頻繁在 Discord 提問和 Release
4. **展現思維**：讓面試官看到你的規劃能力

**頻率**：至少每天 1-2 次提問或 Release

**質量問題範例**：

- 「影片觀看進度是即時儲存還是離開頁面才儲存？」
- 「交付後經驗值是立刻發放還是需要審核？」
- 「排行榜是即時更新還是每日更新？」
- 「道館挑戰的兩種模式（個人/團體）在系統中有什麼差異？」
- 「獎勵任務的機會卡消耗邏輯是什麼？」

### 2. AI 使用技巧

**用 AI 生成**：

- Spring Boot Entity/Repository/Service
- Next.js 頁面 boilerplate
- E2E 測試腳本
- Docker Compose 配置
- 資料庫 Migration 腳本

**自己寫的部分**：

- 核心業務邏輯
- 資料庫 Schema 設計
- API 設計
- 重要的邊界條件處理

### 3. 技術取捨建議（依 PM 確認優先級）

#### ✅ 必須做（Release 1 & 2）

**Release 1 核心**
- 使用者註冊/登入（JWT）
- 課程列表 + 課程詳情
- 影片播放 + 觀看進度追蹤
- 交付功能 + 累積經驗值

**Release 2 核心**
- 購課流程（**簡化版金流**，用按鈕模擬，不串真實支付）
- 購買後解鎖課程

- 複雜的權限系統（基本 JWT 即可）
- 即時通知功能

## 風險管理

### 風險 1：Spring Boot 不熟

**對策**：

- 用 Spring Initializr 快速建立專案
- 讓 AI 寫 80% 的 CRUD
- 只學最基礎的概念
- 重點放在理解 Controller -> Service -> Repository 架構

### 風險 2：E2E 測試沒時間

**對策**：

- Day 2 就建立測試框架
- 每開發一個功能就寫對應測試
- 用 AI 生成測試腳本
- 至少保證核心流程有測試覆蓋

### 風險 3：功能做不完

**對策**：

- **這是預期的！** 他們不期待你做完
- 重點是展現「規劃能力」和「迭代思維」
- 寧可 4 個功能完整，不要 10 個功能半殘
- 確保每個 Release 都是可 Demo 的狀態

### 風險 4：Docker 環境問題

**對策**：

- 盡早測試 Docker Compose
- 準備本地開發和 Docker 兩套環境
- 記錄所有環境設定步驟
